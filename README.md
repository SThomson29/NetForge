# Config Generation Test Fixtures

Each subdirectory is a test scenario run against `ansible-playbook` to verify
config generation. Scenarios cover every supported template feature.

## Scenarios

| Scenario | Switch type | Features tested |
|---|---|---|
| `minimal_l3` | cx | Basic L3: loopback, P2P, access port, SVI, OSPF |
| `full_l2` | cx | VLANs, access/trunk ports, SVIs with DHCP helpers, active gateway, jumbo MTU, multiple VRFs, LAG uplink |
| `ospf_multi` | cx | Multiple OSPF instances, multiple areas, VRF-scoped OSPF, passive SVIs |
| `bgp_ibgp` | cx | iBGP with multiple neighbors, loopback update-source, device role spine |
| `vxlan_evpn` | cx | VXLAN VTEP loopback, VNI map, OSPF underlay, BGP overlay |
| `vsx_primary` | cx_vsx | VSX primary, ISL LAG, keepalive VRF, active gateway SVIs |
| `vsx_secondary` | cx_vsx | VSX secondary role |
| `vsf_stack` | cx_vsf | VSF stacking, 2 members, per-member port numbering |
| `snmp_v3` | cx | SNMPv3 with multiple users, v2c community alongside |
| `aaa_dot1x` | cx | RADIUS (3 servers), dynamic authorization, authenticated ports with default VLAN |
| `static_routes` | cx | Static routes across multiple VRFs with admin distance |
| `lag_variants` | cx | LACP active, passive, static (off), L2 trunk LAG, L3 routed LAG |
| `full_featured` | cx_vsx | Every feature simultaneously: VSX, VXLAN, OSPF multi-instance, iBGP, SNMPv3, RADIUS+dot1x, all interface types, multiple VRFs, static routes, banners, active gateway |

## Structure

```
fixtures/
  <scenario>/
    hosts.ini           # Ansible inventory
    host_vars/
      <hostname>/       # One directory per switch
        *.yml           # All variable files
    expected/
      *.ios             # Golden output files (generated by UPDATE_FIXTURES=1)
```

## Generating golden files

After cloning NetForge and ensuring `ansible-playbook` is on PATH:

It is recommended to use a virtual environment to keep project dependencies
isolated:

```bash
# Generate all golden .ios files from current templates
UPDATE_FIXTURES=1 pytest tests/test_generation.py::test_update_fixtures -v -s

# Commit the results
git add tests/fixtures/
git commit -m "Update generation test golden files"
```

## Running tests

```bash
# All generation tests
pytest tests/test_generation.py -v

# Single scenario
pytest tests/test_generation.py -v -k minimal_l3

# Template smoke tests only (no Ansible required)
pytest tests/test_generation.py::TestJinja2Templates -v

# After a template change — update goldens then run
UPDATE_FIXTURES=1 pytest tests/test_generation.py::test_update_fixtures -v -s
pytest tests/test_generation.py -v
```
